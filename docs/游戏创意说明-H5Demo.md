## 一、为什么是小游戏（而非原生 App / 主机游戏）

### 1.1 碎片化 Session 适配（权重 15%）

| 维度 | 数据 | 说明 |
|------|------|------|
| 单次通关时长 | **10-15 分钟** | 完美匹配通勤/排队/睡前等碎片场景 |
| 可中断节点 | **5个阶段自然断点** | 注册→刷卡→导航→死胡同→终局，每段2-3分钟 |
| 重玩价值 | **中高** | 昵称动态绑定 + 跳水坑结果分支 → 多次体验不同细节 |
| 学习成本 | **接近零** | Swipe / Hold / Drag 三手势仍然直觉，但现在统一由持续拇指引导和底部字幕提示承接，无需单独教程 |

**核心论点**：「替身」的本质是**一段可被"玩"的互动短篇恐怖小说**。它的叙事密度和情绪节奏天然适合碎片化消费——玩家不需要记住复杂系统，每次打开都是一次完整的心理惊悚体验。

### 1.2 社交裂变潜力（权重 15%）

| 裂变触点 | 触发时机 | 分享内容 | 预期转化率 |
|----------|----------|----------|------------|
| **墓碑截图** | 拼图完成瞬间 | "我在这个游戏里看到了自己的名字..." | 🔥🔥🔥 **高**（好奇心驱动） |
| **资料页镜像** | 对方资料更新 | "这个APP刚才记录了我的一举一动..." | 🔥🔥 **中高**（猎奇+恐惧） |
| **终局「已注销」** | 游戏结束黑屏 | 一行字 + 无其他信息 | 🔥🔥 **中**（神秘感引发讨论） |
| **挑战好友** | 通关后弹窗 | "你敢试试吗？输入你的真名..." | 🔥 **中**（社交压力） |

**核心裂变机制设计**：

```
墓碑显现玩家真实昵称 → 截图冲动（强）→ 微信/朋友圈/小红书分享
     ↓
好友看到"自己的名字出现在墓碑上" → 好奇心驱使下载尝试
     ↓
→ 形成病毒式传播闭环（类似《替身》电影口碑效应）
```

**为什么不做成原生 App？**
- **分发门槛**：H5 即点即玩 vs App Store 审核周期 1-3 天
- **试错成本**：H5 可快速迭代叙事节奏 vs 原生发版需重新审核
- **社交传播路径**：微信内直接打开 H5 链接 vs 需跳转 App Store 中断流程
- **开发成本**：纯静态单文件 H5 + 一个验收页即可完成核心演出，仍显著轻于原生 App 至少 2 周的 MVP 成本

## 二、游戏定位与核心玩法

### 2.1 一句话定位

> **「替身」是一款模拟约会 APP 的竖屏心理微恐交互叙事 H5，通过 Tinder 式刷卡 + 地图方向拖拽 + 跳水坑蓄力 + 墓碑拼图三种拇指直觉交互，让玩家体验"被已注销用户逐步接管身份"的心理惊悚过程。**

### 2.2 目标用户画像（权重 10%）

| 维度 | 描述 |
|------|------|
| **核心用户** | 18-30 岁，女性占比 ~60%（悬疑/恋爱/社交类内容消费主力） |
| **次级用户** | 18-35 岁，男性（猎奇/挑战/分享给异性朋友） |
| **行为特征** | 日常使用抖音/小红书/B站、追剧/看短视频、对"细思极恐"类内容有高耐受度 |
| **触达渠道** | 微信朋友圈、小红书笔记、抖音短视频切片、B站实况解说 |
| **付费意愿** | IAA（广告变现）为主，IAP（去广告/多结局DLC）为辅 |

### 2.3 核心玩法三件套

#### 操作层：三大手势

```
┌──────────────────────────────────────────────────────────────────┐
│                      手势 - 场景映射表                           │
├──────────┬────────────────┬────────────────────┬─────────────────────┤
│  手势     │   物理操作      │  游戏场景           │      判定逻辑        │
├──────────┼────────────────┼────────────────────┼─────────────────────┤
│  Swipe   │ 左右滑动屏幕    │ 刷卡匹配            │ X轴位移 >80px       │
│          │                │                    │ 左滑=跳过 右滑=喜欢   │
├──────────┼────────────────┼────────────────────┼─────────────────────┤
│  Hold    │ 长按不松手      │ 跳一跳蓄力          │ touchstart→touchend  │
│          │                │                    │ 时长 360~1100ms 成功 │
├──────────┼────────────────┼────────────────────┼─────────────────────┤
│  Drag    │ 拖拽移动 / 推杆 │ 地图方向选择 / 拼图  │ touchmove 实时更新    │
│          │                │                    │ knob / 残片坐标追踪   │
└──────────┴────────────────┴────────────────────┴─────────────────────┘
```

#### 循环层：五阶段心流曲线

```
焦虑值
  ↑
  │         ╱╲                    ╱╲
  │        ╱  ╲                  ╱  ╲
  │       ╱    ╲        ╱╲     ╱    ╲＿＿＿ 终局
  │      ╱      ╲      ╱  ╲   ╱
  │     ╱  注册   ╲    ╱拼图╲ ╱ 死胡同
  │    ╱           ╲  ╱    ╲╱
  │   ╱   刷卡匹配   ╲╱ 跳水坑
  │  ╱
  │ ╱
  └────────────────────────────────────────────→ 时间
    注册    刷卡     导航事件    失控     终局
```

#### 爽感来源（恐怖设计的"爽"）

| 类型 | 设计手法 | 玩家感受 |
|------|----------|----------|
| **控制幻觉** | 前4个阶段赋予完整操作权 | "我是安全的，我能掌控一切" |
| **剥夺反转** | 死胡同阶段突然禁用所有操作 | "等等...为什么动不了？！" |
| **第四面墙突破** | 资料页精准读取玩家昵称和行为 | "它知道我叫什么...它看到了我刚才做的事" |
| **身份恐惧** | 终局「已 注 销」暗示替换完成 | "我不是我了...那我现在是谁？" |

### 2.4 差异化亮点（权重 10%）

| 维度 | 「替身」 | 传统恐怖游戏 | 恋爱模拟游戏 |
|------|----------|-------------|-------------|
| **恐怖类型** | 心理微恐（无 Jump Scare） | Jump Scare / 追逐战 | 无恐怖元素 |
| **交互深度** | 3种手势 × 5阶段线性叙事 | 复杂操作 / QTE | 文字选项选择 |
| **打破第四面墙** | ✅ 读取玩家真实输入 | ❌ 固定剧本 | ❌ 固定剧本 |
| **重玩性** | 昵称绑定 + 行为分支 | 固定路线 | 多结局分支 |
| **体量** | 10-15 分钟单次 | 2-8 小时 | 3-10 小时 |
| **技术形态** | 纯静态单文件 H5（另带 1 个浏览器验收页） | Unity/Unreal引擎 | Ren'py/Gal框架 |

**一句话差异化**：**它是唯一一款"用约会APP的外壳包装心理惊悚内核"的竖屏H5——让玩家在熟悉的左滑右滑操作中，一步步走进自己无法逃脱的命运。**

## 三、竞品分析与市场机会

### 3.1 市场窗口判断

| 趋势信号 | 数据来源 | 与「替身」的关联 |
|----------|----------|------------------|
| 短剧/微恐内容爆发 | 抖音#细思极恐 播放量 50亿+ | 目标用户重叠度 >70% |
| 竖屏交互叙事兴起 | 《隐形守护者》Steam 200万套 | 验证了竖屏叙事的市场接受度 |
| H5营销游戏常态化 | 品牌H5活动月均 200+ 个 | 技术成熟，分发渠道畅通 |
| 恐怖游戏女性用户增长 | Steam女性玩家占比升至 34% | 目标核心用户持续扩大 |
| "替身/分身"话题热度 | 微博#替身 阅读 3.2亿 | 社交媒体自带流量池 |

**结论**：**当前是竖屏微恐叙事H5的蓝海窗口期**——传统恐怖游戏太重（引擎/下载门槛），而轻量H5市场缺乏高品质叙事产品。「替身」恰好填补这一空白。

### 3.2 竞品交叉分析

#### 直接竞品（竖屏恐怖/悬疑 H5）

| 产品 | 优势 | 劣势 | 替身的攻击面 |
|------|------|------|-------------|
| **《纸嫁衣》H5试玩版** | IP知名度高、美术精良 | 需要下载APP、体量过大(>100MB) | **即点即玩、零下载** |
| **《流言侦探》** | 文字推理深度优秀 | 纯文字无触控交互、学习成本高 | **直觉手势、零学习成本** |
| **微信恐怖解谜H5** | 分发便捷 | 广告体验差、剧情模板化 | **精品化叙事、无广告干扰(MVP)** |

#### 间接竞品（占用相同用户时间的应用）

| 产品类型 | 用户重叠度 | 替身的差异化突围策略 |
|----------|-----------|---------------------|
| 抖音/快手短视频 | ★★★★★ | **主动交互 vs 被动观看**——玩家是"参与者"不是"观众" |
| 乙女/恋爱手游 | ★★★★☆ | **反套路叙事**——表面像恋爱APP，实际是恐怖故事 |
| 微信小程序游戏 | ★★★☆☆ | **沉浸式全屏体验**——非小程序沙盒限制 |

### 3.3 用户情绪锚点

```
目标用户的情绪需求光谱：

无聊 ──→ 好奇 ──→ 紧张 ──→ 恐惧 ──→ 释放 ──→ 分享欲
  ↑                                              │
  └── 「替身」在此处切入 ─────────────────────────┘
       (用熟悉的手势建立安全感)
           (用异常卡片制造好奇)
               (用导航移动积累紧张)
                   (用墓碑/资料页触发恐惧)
                       (用终局留白实现释放)
                           (用"看到自己名字"激发分享)
```

## 四、商业化路径（权重 20%）

### 4.1 变现模型：IAA + IAP 混合

| 层级 | 变现方式 | 触发时机 | 预期 ARPU |
|------|----------|----------|-----------|
| **L1: 免费体验** | 激励视频广告 | 通关后看广告解锁"隐藏真相"片段 | ¥0.3-0.5/次 |
| **L2: 去广告** | IAP ¥6 | 永久去除广告 | ¥6/人 |
| **L3: 多结局 DLC** | IAP ¥12 | 解锁"替身的视角"/"第三人称视角"等新剧本 | ¥12/人 |
| **L4: 品牌联名** | 定制化品牌H5 | 品牌植入（如"某奶茶店"作为导航途经点） | ¥10-50万/次 |

### 4.2 关键指标预期（MVP 后 3 个月）

| 指标 | 保守估计 | 乐观估计 | 行业基准 |
|------|---------|---------|---------|
| 日活跃用户(DAU) | 5,000 | 50,000 | - |
| 次日留存 | 25% | 40% | 休闲H5均值 20% |
| 7日留存 | 10% | 20% | 休闲H5均值 8% |
| 广告渗透率 | 30% | 50% | 行业均值 35% |
| 社交分享率 | 10% | 25% | 优质内容 >15% |
| 付费转化率 | 2% | 5% | IAP均值 3% |

### 4.3 商业化风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 广告影响沉浸体验 | 用户流失 | 广告仅在"安全区"（通关后/切换阶段前）展示 |
| 恐怖内容审核不通过 | 无法上架主流平台 | 预留"温和版"（降低恐怖强度）供审核使用 |
| 病毒传播生命周期短 | DAU快速衰减 | 持续推出新剧本/节日限定版本维持热度 |

## 六、总结

### 「替身」的核心竞争力公式

```
                    熟悉的操作外壳（Tinder刷卡）
                           +
                    惊悚的叙事内核（身份替换）
                           +
                    打破第四面墙（读取玩家输入）
                           =
    一款"让人忍不住截图分享"的竖屏心理微恐H5
```

### 为什么现在是最佳时机？

1. **内容趋势**：微恐/细思极恐内容在短视频平台爆发式增长
2. **技术成熟**：H5性能足以支撑流畅的触控交互体验
3. **分发红利**：微信/抖音生态为H5提供零成本触达通道
4. **市场空白**：轻量级高品质叙事H5供给不足，「替身」填补缺口
5. **社交货币属性**：墓碑显示玩家名字 = 天然的自传播钩子
