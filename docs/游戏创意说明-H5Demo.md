# 《替身》H5 Demo 创意说明

**文档版本：v1.3.0**

## 一、项目定位

《替身》是一款以“被替换感”为核心体验的短流程惊悚 H5。玩家从熟悉的约会匹配开始，被逐步引向一条越来越不对劲的路线：不是单纯被吓到，而是不断感到**自己的操作权正在被夺走**。

## 二、核心体验目标

- **从日常滑入异常**：前半段足够像普通匹配 App，让异常更刺。
- **交互本身制造压迫**：不是靠说明文字吓人，而是让滑卡、导航、擦拭、权限弹窗这些熟悉动作逐渐失控。
- **终局强调“被接管”**：最后不是玩家做选择，而是玩家眼看着设备被“替身”代替自己使用。

## 三、当前流程（v1.3.0）

### 3.1 匹配阶段

- 玩家输入名字后进入资料卡界面。
- 前 3 张是正常用户。
- 第 4 张是纯黑异常用户。
- 当玩家尝试左滑异常卡时：
  - 卡片弹回原位
  - 弹出内心独白：`这个人还怪特别的，不如匹配一下试试...`
- 玩家只能右滑，随后进入 Tinder 式匹配动画。

### 3.2 对话阶段

- 匹配成功后进入聊天页。
- 对方发送地址链接，并附文：`来找我。`
- 玩家点击地址链接，切入地图 App。

### 3.3 阶段三：地图与事件

#### 地图导航

- 采用 **上半 AR 实景 + 下半 2D 地图** 的双层结构。
- AR 区域显示真实街景参考图、蓝色引导路径与大字号距离。
- 玩家在上半屏按住并向上推，才会持续前进；手一松开，行走立刻停止。
- 屏幕中有常驻手势剪影演出，AR 层里还有跟手拇指压屏效果。

#### 事件一：跳一跳

- 情景：前方出现水坑。
- 操作：长按按钮蓄力，松开跳跃。
- 结果：
  - 力度适中：`完美跳过水坑！`
  - 力度过大/过小：`哎呀，踩到水坑里了...`
- 该结果会被记录，作为气氛反馈的一部分。

#### 事件二：擦拭路牌

- 情景：前方出现被泥污遮住的路牌。
- 操作：玩家在牌面连续擦拭。
- 要求：`不要回头` 四个字在开始时必须完全不可见。
- 只有当擦拭完成后，文字才会显现。
- 完成后震动一次，并弹出强烈独白：
  - `这是...这是什么鬼！！！我得离开这里！`

### 3.4 资料更新

- 路牌事件结束后，顶部滑下通知：`【未知用户】资料更新了，快去看看`
- 进入资料页后，签名会改写为带有玩家名字的异常内容。
- 距离会持续缩短，直到归零。
- 这一段必须保持约会 App 的界面外壳，只是内容越来越诡异。

### 3.5 终局：The Replacement

#### 异化权限请求

- 距离归零后，地图消失，切入系统弹窗。
- 文案：`{玩家名称} 想要访问你的相机，以完成同步。`
- 只有一个 `允许` 按钮，并附带 3 秒倒计时。
- 无论玩家主动点击，还是倒计时结束，都会进入下一步。

#### 寻找宿主（伪造取景框）

- 画面切为纯黑取景框界面。
- 人脸识别框会游移，最后在屏幕中央锁定并变绿。
- 下方浮现：
  - `[已识别：{玩家名称}]`
  - `[状态：适合接管]`

#### 视界交替与彻底失控

- 取景框像老电视一样向中心收缩息屏。
- 短暂黑屏后，亮起一个极其日常的伪造手机桌面。
- 顶部弹出微信新消息：`妈妈：你到哪了？天黑了快点回来。`
- 一只“替身之手”从画面边缘伸出，自行点击通知。
- 画面切入聊天界面，这只手自动输入：`快到了，在路上。`
- 它自行点击发送，玩家无法干预。
- 随后画面切黑，标题 `替 身` 缓缓淡入。

## 四、视觉原则

- 前半段保持克制，不要过早出现夸张惊悚符号。
- 内心独白底图必须明显区分于背景，像电影字幕黑条，而不是轻飘飘的一层字。
- 地图阶段要偏写实：街景、蓝色 AR 路径、距离数字、手势压屏都要让人相信“这是一个真的导航界面”。
- 终局阶段要强调熟悉感被劫持：桌面、微信、发送按钮都看起来越正常越好。

## 五、当前实现说明

- 已移除旧引导文案与旧墓碑段。
- 不调用真实相机权限，终局完全使用伪造界面演出。
- 手部素材来自用户提供的参考图，并在运行时自动去掉浅色底，只保留黑色手势轮廓。
- 验收页 `test.html` 已同步到 v1.3 口径。
