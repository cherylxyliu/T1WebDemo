## 核心玩法循环

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌──────────┐    ┌──────────┐
│  ① 注册     │ → │  ② 刷卡匹配 │ → │  ③ 导航赴约 │ → │  ④ 死胡同│ → │  ⑤ 终局  │
│  输入昵称   │    │  写实资料卡 │    │  方向拖拽   │    │  操作失效│    │  已注销  │
│  验证码注册 │    │  强制匹配   │    │  相机/拼图  │    │  距离失控│    │  身份替换│
└─────────────┘    └─────────────┘    └─────────────┘    └──────────┘    └──────────┘
                         ↓                   ↓
                  底部独白字幕条      跳一跳 / 相机 / 墓碑拼图
                  持续拇指引导        结果影响后续资料页
```

### 三大核心手势

| 手势 | 操作 | 应用场景 | 判定逻辑 |
|------|------|----------|----------|
| **Swipe** | 左右滑动 | 刷卡匹配 | touchstart ↔ touchend 的 X轴差值 |
| **Hold** | 长按蓄力 | 跳水坑 | 按压时间差 (ms)，0.36s~1.1s 为成功区间 |
| **Drag** | 拖拽方向 / 拖拽拼图 | 地图移动 / 墓碑拼图 | touchmove 实时更新 knob 或残片位置 |

### 这次 v1.1.0 重构补了什么

- 底部黑色电影字幕条，独立承载内心独白，不再压住资料卡内容

- 前几张约会资料卡改成更写实的人设与伪照片，异常卡的违和感更晚才暴露

- 新增明确的相机权限剧情节点，并对不支持 `getUserMedia` 的打开方式做降级演出
- 墓碑拼图增加目标轮廓、错误回弹、吸附完成和完整碑文显现反馈

## 爽点与恐怖感设计

### 从"探索"到"被困"的反转
前半段赋予玩家强控制感（左右滑选择、方向拖拽、主动拼图、自己决定是否开相机），在死胡同阶段**剥夺所有操作权限**，制造失控心理落差。

### "强迫中奖"的异常匹配
异常卡片无法左滑——违背 Tinder 类应用"双向选择"的常识，暗示**命运的不可违逆**。

### 打破第四面墙的窥视
资料页精准读取玩家的真实昵称与即时操作表现（跳水坑成功/失败），产生**"屏幕里的东西正在看着我玩"**的错觉。

### 身份替换的终极恐惧
不是 Jump Scare（跳脸惊吓），而是发现**"我正在慢慢变成那个注销用户"**。终局的「已 注 销」完成心理闭环。

## 故事节点详解

### 阶段一：注册（建立代入感）
- 手机号预填 `138****7291`（灰色禁用态）
- 获取验证码 → SMS Toast 通知滑入
- **核心输入：玩家昵称**（后续传递给墓碑 + 资料页）

### 阶段二：刷卡匹配（Tinder-like）
- 3张更写实的正常资料卡（金融男 / Livehouse 主理人 / 摄影师）+ 底部独白字幕条
- 第4张异常卡片：**全黑照片 / 昵称「已注销用户」/ 签名「来找我。别再左滑了。」**
- 左滑弹回 + 独白提示 + 持续拇指引导右滑 → 匹配成功

### 阶段三：导航移动 + 交互事件
- 深色地图 + 蓝色路径线 + **底部固定方向钮拖拽**
- **事件A - 跳一跳**：长按蓄力跳跃水坑（结果被记录）
- **事件B - 相机权限**：对方要求你打开相机确认位置（支持时真实请求，不支持时降级演出）
- **事件C - 拼图墓碑**：拖拽4块碎片到对应槽位 → 吸附完成后显现「{玩家名}之墓」

### 阶段四：死胡同（失控开始）
- 对方资料更新通知 → 头像变玩家轮廓 / 签名记录跳水坑结果
- 重新规划路线 → 移动50m后停止 → **「您已到达目的地。前方无路可走」**
- 距离数字失控递减：120m → 87m → 53m → ... → 0m

### 阶段五：终局（The Replacement）
- 纯黑背景 + 1-2秒留白
- 「**已 注 销**」白色文字缓缓淡入（字间距4px，动画2s）
- **暗示：玩家身份已被彻底抹除，替换完成**

## 技术规格

| 维度 | 规格 |
|------|------|
| 开发方式 | 纯 HTML + CSS + Vanilla JavaScript（单文件） |
| 依赖项 | 零外部依赖（无需 npm/webpack/vite） |
| 屏幕比例 | 9:16 竖屏锁定 |
| 触控事件 | touchstart / touchmove / touchend + mousedown / mousemove / mouseup |
| 音频 | ❌ 当前仍无音频 |
| 相机权限 | ✅ 已补剧情节点；支持 `getUserMedia` 时可真实请求，其他环境降级演出 |

| 验证方式 | 打开 `test.html` 可自动检查 7 个关键交互点 |
