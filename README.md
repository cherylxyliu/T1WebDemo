## 核心玩法循环

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌──────────┐    ┌──────────┐
│  ① 注册     │ → │  ② 刷卡匹配 │ → │  ③ 导航移动 │ → │  ④ 死胡同│ → │  ⑤ 终局  │
│  输入昵称   │    │  Tinder滑动 │    │  长按前进   │    │  操作失效│    │  已注销  │
│  验证码注册 │    │  强制匹配   │    │  触控事件   │    │  距离失控│    │  身份替换│
└─────────────┘    └─────────────┘    └─────────────┘    └──────────┘    └──────────┘
                         ↓                   ↓
                   异常黑色卡片      跳一跳 / 拼图墓碑
                   无法左滑弹回      结果影响后续剧情
```

### 三大核心手势

| 手势 | 操作 | 应用场景 | 判定逻辑 |
|------|------|----------|----------|
| **Swipe** | 左右滑动 | 刷卡匹配 | touchstart ↔ touchend 的 X轴差值 |
| **Hold** | 长按蓄力 | 地图移动 / 跳一跳 | 按压时间差 (ms)，0.3s~1s 为成功区间 |
| **Drag** | 拖拽定位 | 拼图墓碑 | touchmove 实时更新 left/top 坐标 |

## 爽点与恐怖感设计

### 从"探索"到"被困"的反转
前半段赋予玩家强控制感（左右滑选择、长按走路、主动拼图），在死胡同阶段**剥夺所有操作权限**，制造失控心理落差。

### "强迫中奖"的异常匹配
异常卡片无法左滑——违背 Tinder 类应用"双向选择"的常识，暗示**命运的不可违逆**。

### 打破第四面墙的窥视
资料页精准读取玩家的真实昵称与即时操作表现（跳水坑成功/失败），产生**"屏幕里的东西正在看着我玩"**的错觉。

### 身份替换的终极恐惧
不是 Jump Scare（跳脸惊吓），而是发现**"我正在慢慢变成那个注销用户"**。终局的「已 注 销」完成心理闭环。

## 故事节点详解

### 阶段一：注册（建立代入感）
- 手机号预填 `138****7291`（灰色禁用态）
- 获取验证码 → SMS Toast 通知滑入
- **核心输入：玩家昵称**（后续传递给墓碑 + 资料页）

### 阶段二：刷卡匹配（Tinder-like）
- 3-4张正常卡片（自由左右滑 + 内心独白）
- 第5张异常卡片：**全黑照片 / 昵称「已注销」/ 签名「来找我。」**
- 左滑弹回 + 强制内心独白引导右滑 → 匹配成功

### 阶段三：导航移动 + 交互事件
- 深色地图 + 蓝色路径线 + 长按移动按钮
- **事件A - 跳一跳**：长按蓄力跳跃水坑（结果被记录）
- **事件B - 拼图墓碑**：拖拽4块碎片 → 显现「{玩家名}之墓」

### 阶段四：死胡同（失控开始）
- 对方资料更新通知 → 头像变玩家轮廓 / 签名记录跳水坑结果
- 重新规划路线 → 移动50m后停止 → **「您已到达目的地。前方无路可走」**
- 距离数字失控递减：120m → 87m → 53m → ... → 0m

### 阶段五：终局（The Replacement）
- 纯黑背景 + 1-2秒留白
- 「**已 注 销**」白色文字缓缓淡入（字间距4px，动画2s）
- **暗示：玩家身份已被彻底抹除，替换完成**

## 技术规格

| 维度 | 规格 |
|------|------|
| 开发方式 | 纯 HTML + CSS + Vanilla JavaScript（单文件） |
| 依赖项 | 零外部依赖（无需 npm/webpack/vite） |
| 屏幕比例 | 9:16 竖屏锁定 |
| 触控事件 | touchstart / touchmove / touchend |
| 音频 | ❌ MVP阶段无音频 |
| 传感器 | ❌ 无陀螺仪/相机/麦克风 |
| iOS适配 | apple-mobile-web-app-capable + viewport-fit=cover + safe-area-inset |
