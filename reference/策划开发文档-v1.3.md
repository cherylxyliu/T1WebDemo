# 《替身》策划开发文档归档

**归档版本：v1.3.0**

## 一、本轮改版背景

v1.2 完成了异常匹配、聊天跳地图、资料更新与伪终局的基础骨架，但用户明确指出以下问题：

1. 旧权限弹窗后无法形成可信的“假取景框接管”体验。
2. 阶段三导航效果差，底部拖拽没有真实推进感。
3. 擦拭路牌时 `不要回头` 提前可见，破坏了揭示感。
4. 终局不够像“替身接管手机”，更像简单黑屏收尾。

因此 v1.3 的目标不是微调，而是**重做地图交互与终局演出**。

## 二、用户最终确认的 v1.3 目标

### 2.1 地图阶段

- 导航改为 **上半 AR 实景 + 下半 2D 地图**。
- 使用真实街景参考图作为 AR 背景。
- 玩家在 AR 区域按住并向上推，才会持续前进；距离必须实时减少。
- 大拇指不能再是小图标或教程提示，而是跟手移动、带按压感的写实手势演出。

### 2.2 路牌事件

- 不能一开始就看见 `不要回头`。
- 玩家要先擦掉泥污，文字完成揭示后才出现。

### 2.3 终局阶段

必须改成下面这条链路：

1. 单按钮权限请求：`{玩家名称} 想要访问你的相机，以完成同步。`
2. 3 秒倒计时，无论点击还是倒计时结束都继续。
3. 纯黑取景框 + 对焦框游移 + 锁定变绿 + `[已识别]` / `[适合接管]`
4. 老电视式息屏
5. 伪造桌面
6. 微信通知：`妈妈：你到哪了？天黑了快点回来。`
7. 替身之手代替玩家点开通知、输入并发送 `快到了，在路上。`
8. 黑屏标题 `替 身` 淡入

## 三、资源落地

本轮已把用户给的参考素材纳入仓库：

- `assets/street-ar.jpg`：AR 街景参考图
- `assets/hand-point.png`：指向上手势，用于地图拇指与匹配页常驻演出
- `assets/hand-palm.png`：张开手掌，用于终局里的“替身之手”

补充处理：

- 两张手图原始文件带浅灰底，不适合直接贴图。
- 运行时通过 Canvas 自动抠底：去掉浅色背景，只保留黑色手势轮廓，保证移动端和 GitHub Pages 一致。

## 四、代码结构调整

### 4.1 文件拆分

v1.2 使用单文件结构，不利于继续改大段交互。本轮改为：

- `index.html`：页面结构
- `styles.css`：界面样式
- `app.js`：流程控制、事件逻辑、测试 API

### 4.2 保留的核心流程

- 玩家名输入
- 3 张正常资料卡
- 1 张异常黑卡
- 左滑异常卡弹回 + 指定独白
- 右滑异常卡进入匹配动画
- 聊天页地址链接跳地图
- 资料更新页距离缩短到零

### 4.3 重做的模块

- 旧地图拖拽步进逻辑 → 改为上推持续前进
- 旧终局权限弹窗与假相机 → 改为完整接管终局
- 旧路牌直接可见文字 → 改为完全隐藏后揭示

## 五、验证策略

本轮先改 `test.html`，再做实现，保证所有核心修改都有自动验收标准。

当前验收结果：**PASS 10/10**

覆盖内容：

- 独白层显示
- 写实手势素材存在且无旧引导气泡
- 异常卡左滑弹回
- 异常卡右滑进入聊天
- 地图页 AR + 2D 结构
- 持续上推前进导致距离缩短
- 路牌文字默认隐藏，擦完才显现
- 不调用真实相机
- 单按钮权限 → 锁定识别 → 伪桌面 → 替身之手接管 → 标题淡入

## 六、后续维护提醒

- 后续若继续改终局，必须优先保护“玩家失去操作权”这个核心，不要退回普通跳脸惊吓。
- 如果再换手部素材，需同步确认是否仍需要运行时抠底。
- 文档更新时，必须同时检查：**标题版本、正文内容、底部落款** 三者一致。
